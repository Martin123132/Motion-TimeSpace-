## 3. Why Amplitude First?

The amplitude problem touches several pillars at once:

- cosmology memory/load normalization;
- the `u3` or quarter-lock style factors;
- `p=3` style exponent/volume arguments;
- `B_mem` or equivalent memory amplitude;
- whether closure hits are derived, fitted, or numerological.

If amplitude is not derived or bounded, empirical successes can be too easy to fake.

So the next gate is:

```text
Any amplitude factor must be derived, bounded, or explicitly demoted to fitted closure.
```

No mystery normalization gets treated as a theorem.

## 4. Why Not Empirical First?

Empirical tests are still essential.

But after checkpoint 80, empirical testing would test:

```text
diagnostic closures
```

not:

```text
derived local field theory
```

That is allowed, but it must be labelled correctly.

The sequence is therefore:

```text
amplitude discipline -> empirical closure testing -> perturbation/lensing/growth contract
```

## 6. Guardrails Going Forward

| Guardrail | Rule |
|---|---|
| no public local-GR claim | local GR remains not derived |
| closures are allowed but labelled | `C_coh` may be tested as diagnostic closure |
| baseline fairness | empirical tests must include appropriate GR/LambdaCDM or other baselines |
| amplitude honesty | amplitudes must be derived, bounded, or fitted |
| no hidden selector reentry | do not revive `C_coh` parent route without a new mechanism |

This also answers a concern from earlier work:

```text
If MTS gets jackknifed or stress-tested, baselines must be subjected to comparable tests where meaningful.
```

Otherwise we cannot tell whether the test broke the theory or the pipeline.
